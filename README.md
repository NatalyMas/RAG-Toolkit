# RAG-Toolkit

Сервис для работы с векторной базой данных Qdrant, реализующий RAG (Retrieval-Augmented Generation) функциональность. Предназначен для интеграции в конструктор задач как набор готовых функций-кирпичиков для построения сценариев обработки документов. Компоненты заменяемые — если завтра админ захочет использовать новую модель, достаточно поменять пути в конфиге.

### Возможности
- Управление коллекциями: создание, удаление, получение информации
- Работа с документами: добавление, поиск (семантический и по метаданным), удаление
- Пакетная обработка: добавление нескольких документов за раз
- Валидация запросов: фильтрация через LLM (Mistral/Ollama)
- REST API: все функции доступны через HTTP endpoints

### Компоненты архитектуры
- **FastAPI** — REST API слой. Принимает HTTP запросы, маршрутизирует к RAG Manager
- **RAG Manager** — оркестратор. Координирует работу всех сервисов, управляет коллекциями и документами
- **Qdrant** — векторная база данных. Хранит эмбеддинги и метаданные, выполняет семантический поиск
- **Embedding Service** — сервис эмбеддингов. Преобразует текст в векторы, используется для индексации и поиска
- **Ollama (Mistral)** — валидация запросов. Фильтрует запросы по заданным критериям (опционально)
